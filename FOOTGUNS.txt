- If an interface does not have anything `inline_for_extraction`,
  it will not be traversed, and inlining will fail. I think this
  even happens if the `inline_for_extraction` does appear but is
  only on a Pulse definition. You can add
  `inline_for_extraction let () = ()` to just force it.

- For inlining, pretty much every concrete function must be
  `inline_for_extraction noextract`. Only kernels *must* be
  top-level, and with a __global__. Everything else is probably
  safe to inline. Note: this applies to typeclasses instances too,
  if they are used concretely (e.g. clayout) they must be
  `inline_for_extraction noextract`.

- If a type has a concrete parameter, like `foo (a : nat)`, we
  can use foo over an erased nat most of the time. But, trying
  to project a field out of it will bump the effect to ghost,
  as the projector has a concrete `nat` parameter instead of erased.
  This is probably worthwhile to fix in F*.

- Do not write `erased (natlt z)`, ideally don't write erased of
  any refined type. Since erased is invariant wrt types this makes
  typechecking much more brittle and really hard to debug.
  Instead refine the erased value like:
    n:(erased nat){n < z}
  or use the alias `enatlt` for this particular one.
